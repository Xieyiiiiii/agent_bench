## Core algorithm passage, reduced to pseudocode

```text
input: query terms q[0..Q), postings per term, doc_len[], idf[], k
initialize score_q8[doc] = 0 and active[doc] = false
for each term t in q:
    postings = posting_list[t]
    if postings is empty:
        count empty_terms
        continue
    for each (doc_id, tf) in postings:
        if metadata filter rejects doc_id:
            count filtered_out
            continue
        active[doc_id] = true
        score_q8[doc_id] += bm25_term_score_q8(t, doc_id, tf)
for each active doc:
    update max-score top-k
return top-k scores and counters
```

## Notes

Haystack establishes retriever shape and filter/top-k flow. The v0 scoring
formula is explicitly BM25Okapi-style and comes from `rank_bm25`; it must not be
described as a complete Haystack BM25L implementation.

For v0, `idf_q8[]` is deterministic synthetic input generated by `init_index()`.
The benchmark does not implement `BM25Okapi._calc_idf()`, negative-IDF epsilon
flooring, tokenizer behavior, or corpus preprocessing. This keeps the CGRA
kernel focused on posting-list traversal, metadata branches, fixed-point term
scoring, score accumulation, and active-document Top-K collection.

## Benchmark-only extensions

- Deterministic query term IDs replace tokenization.
- Deterministic posting lists, document lengths, metadata, and `idf_q8[]`
  replace corpus preprocessing.
- BM25 arithmetic uses Q8 fixed-point integers instead of Python floats.
- `avg_doc_len` is a documented synthetic constant in v0. If a future benchmark
  computes it from `doc_len[]`, update this reference before changing C behavior.

## Fixed-point scoring contract

All BM25 arithmetic in C must use Q8 integers and int64 intermediates:

```text
Q8_ONE = 256
q8_mul(a_q8, b_q8) = (a_q8 * b_q8) / Q8_ONE
q8_div(numer_q8, denom_q8) = (numer_q8 * Q8_ONE) / denom_q8

norm_q8 = Q8_ONE - b_q8 + (b_q8 * doc_len[doc_id]) / avg_doc_len
denom_q8 = tf * Q8_ONE + q8_mul(k1_q8, norm_q8)
tf_num_q8 = tf * (k1_q8 + Q8_ONE)
tf_weight_q8 = q8_div(tf_num_q8, denom_q8)
term_score_q8 = q8_mul(idf_q8[t], tf_weight_q8)
score_q8[doc_id] += term_score_q8
```

Division truncates toward zero. If `avg_doc_len` or `denom_q8` is zero, the
term contribution must be zero and the benchmark should still complete.

## C function mapping contract

```text
initialize postings/doc stats      -> init_index
initialize query terms             -> init_query
for each query term                -> run_kernel
posting list traversal             -> score_posting_list_q8
metadata filter rejects doc        -> passes_filter
BM25Okapi term contribution        -> bm25_term_score_q8
active document final selection    -> collect_active_docs_into_topk
```

Only `bm25_term_score_q8` should implement the scoring formula. Other functions
should orchestrate traversal, filtering, and Top-K collection.

## Behavior matching constraints

- Empty posting lists must increment `EMPTY_TERMS`.
- Filtered postings must not update `score_q8[]` or `active[]`.
- A document becomes active only after at least one accepted posting contributes
  to its score.
- Final Top-K is collected only from active documents.
- `idf_q8`, `score_q8`, and intermediate fixed-point values must use `_q8`
  suffixes.
- Q8 multiplication, division, rounding, and zero-denominator behavior must
  match the fixed-point scoring contract above.
